# 📄 Project Charter: The QEC Code Showdown

**Benchmarking Quantum Error Correction Against Realistic Hardware Noise**

### 🎯 The Objective

To move beyond basic theoretical quantum circuits and empirically benchmark four distinct Quantum Error Correction (QEC) algorithms. We will test these algorithms against simulated real-world hardware noise using Qiskit Aer to determine which architecture is the most robust.

We will also address the mathematical scaling limitations of $t=1$ error correction to show a comprehensive understanding of current quantum computing challenges.

------

### 🏟️ The Shared Arena (Crucial for Integration)

To ensure our final data is comparable, **all four algorithms must be built to the exact same specifications.** We cannot have four disconnected scripts.

- **The Framework:** Qiskit.
- **The Noise Model:** We will use `qiskit_aer.noise` combined with `NoiseModel.from_backend()` to fetch real calibration data from an IBM backend (e.g., `ibm_kyoto` or `ibm_brisbane`).
- **Input Standard:** Every circuit must accept a single, arbitrary logical qubit state (e.g., $|\psi\rangle = \alpha|0\rangle + \beta|1\rangle$).
- **Output Standard:** Every circuit must output a corrected logical qubit state so the Data Scientist can calculate the fidelity (success rate) against the original input.

### 📅 Project Milestones & Timeline

- **Phase 1: Local Builds:** Everyone builds and tests their specific algorithm in a *noiseless, ideal* Qiskit simulator to ensure the logic and syndrome measurements work perfectly.
- **Phase 2: Arena Integration:** Member 1 finalizes the noise script. Everyone integrates their circuits into the master file.
- **Phase 3: Data Harvesting:** Run thousands of "shots" across the simulated IBM backend for all four codes at varying noise levels.
