Perfect [[5,1,3]] Quantum Error Correction

Protect one logical qubit using five physical qubits, then test when error correction helps.

The problem

Quantum information is fragile

  • Bit flips change computational basis states.
  • Phase flips damage superpositions.
  • Gates, measurement, decoherence, and crosstalk all add noise.
Long computation + noisy hardware = lost quantum state
The quantum twist

We cannot simply copy a qubit

Classical repetition copies a bit. Quantum states cannot be cloned.

QEC instead spreads one logical state across entangled physical qubits.

|psi> = alpha|0> + beta|1>

|psi_L> = alpha|0_L> + beta|1_L>
The process

Encode, detect, correct

The syndrome tells us what kind of error happened without measuring the logical information itself.

The code

Why the five-qubit code matters

  • n=5: physical qubits
  • k=1: logical qubit
  • d=3: code distance
  • t=1: one arbitrary error corrected
Smallest arbitrary single-qubit QEC code

Theory

Expected error rates

No QEC: 2p/3
Perfect code: 1 - (1-p)^5 - 5p(1-p)^4
Results

Physical error vs logical error

We sweep p=0 to p=0.05, run 1000 shots at each point, and compute logical error.

Threshold

When does QEC stop helping?

Below the crossing, QEC helps. Above it, overhead and multi-error events dominate.

p ~= 0.13755 one-round crossing estimate
Usefulness

Best use case

The Perfect code is most useful when qubits are scarce and physical error is low enough that two-error events are rare.
